## 三、安全性审计

### 🔴 HIGH 严重问题 (3)

| # | 问题 | 位置 | 风险评估 |
|---|------|------|---------|
| 1 | **API Keys 硬编码** | `start-aigc.sh`: `ALIYUN_API_KEY="sk-..."` | 🔴 凭据明文存储，Git 历史可回溯 |
| 2 | **无速率限制** | 后端未实现 rate limiting | 🔴 注入/爆破/DoS 风险 |
| 3 | **无 CSRF/XSS 防护** | 后端未使用 helmet 或类似中间件 | 🔴 跨站攻击面完全暴露 |
| 4 | **MinIO 默认口令** | docker-compose: `minioadmin/minioadmin` | 🔴 对象存储可任意读写 |

### 🟡 MEDIUM 问题 (5)

| # | 问题 | 位置 |
|---|------|------|
| 5 | 前端 API Token 存 localStorage | `useStudioStore.ts` 使用 `localStorage.getItem('token')` |
| 6 | 文件上传无大小/类型校验 | 后端无 `@fastify/multipart` 限流配置 |
| 7 | 仅单层 CORS 配置 | cors 插件已注册，但无白名单限制 |
| 8 | 无 SQL 注入显式防护 | 依赖 Prisma 的参数化查询为隐式保护 |
| 9 | 生产环境日志中含有敏感信息 | `CRYPTO_ENCRYPTION_KEY` 长度在启动时打印 |

### 安全建议
1. **立即删除启动脚本中的明文 API Key** → 使用环境变量文件 `.env`（已在 `.gitignore`）
2. **添加 rate limiting**: `@fastify/rate-limit`
3. **添加安全头**: `@fastify/helmet`
4. **MinIO 更换密钥**，生产环境使用 IAM

## 六、性能审计

| 指标 | 评估 | 说明 |
|------|------|------|
| 前端构建产物 | 🟢 3.3MB | Nuxt SPA 模式，合理 |
| 编辑器内存 | 🟡 中风险 | blob URL 无上限，大量上传可能 OOM |
| 后端启动 | 🟢 正常 | PM2 管理，启动脚本含 env 注入 |
| 数据库 | 🟢 正常 | PG 16 容器化，178 表，schema 3,423 行 |
| 视频合成 | 🟡 降级 | FFmpeg.wasm 未安装，走模拟导出 |

#### 建议
1. **编辑器素材上限**：建议对 blob URL 数量做上限（如最多 50 个文件）
2. **安装 FFmpeg.wasm**：`npm install @ffmpeg/ffmpeg` 启用真实浏览器端导出
3. **视频缩略图懒加载**：大量上传时首屏只加载可见部分的缩略图
4. **添加 Service Worker**：缓存 editor 静态资源，减少初次加载
